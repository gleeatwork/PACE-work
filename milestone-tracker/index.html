Notes & tips
How dates get set: click Finish date on any milestone to pin it to a fixed, external date (shown in dark teal) — use this for things like grant deadlines, regulatory dates, or contract dates you already know. Leave it "auto" and it shows the computed date instead — set to match whatever depends on it starting on time. Latest Start is that finish date minus the milestone's own Duration.
About the CRITICAL badge: it marks the tightest chain of milestones feeding the actual final deadline. A milestone with its own fixed date that other things depend on (like an early internal submission deadline) is evaluated against its own date, not folded into the main critical-path comparison — so a milestone that's individually tight but isn't the true bottleneck for the final deadline won't misleadingly steal the badge.
This is a generic template. The engine (this HTML file) is shared across every workstream — only the data changes.
Workstream tabs: the sidebar switches between separate JSON files (enrollment.json, um.json, etc.) living in this same folder. Export always downloads the file matching the tab you're on, ready to overwrite that same file in GitHub.
Adding a new workstream: use one of the + Blank slot tabs — type a real name into the Project Title field above, add milestones, Export. Once all blank slots are used, ask Claude to add more (one line of code each).
Persistence: edits live in this browser tab only until you Export. Export → commit the JSON to your GitHub repo → next person who opens that tab gets your latest version.
